1. After install, retrieve Redis passwords:

   # Front Redis password (bitnami chart stores password in a secret named <release>-redis-front by default,
   # but check the chart's secret name if you changed fullnameOverride)
   kubectl get secret --namespace <namespace> <release>-redis-front -o jsonpath="{.data.redis-password}" | base64 --decode

   # Back Redis password
   kubectl get secret --namespace <namespace> <release>-redis-back -o jsonpath="{.data.redis-password}" | base64 --decode

2. Connect from within the cluster (example using redis-cli pod):

   kubectl run --namespace <namespace> -i --tty redis-client --image=bitnami/redis:latest --rm -- bash
   # For front
   redis-cli -h <service-name-for-front> -a <front-password>
   # For back
   redis-cli -h <service-name-for-back> -a <back-password>

3. Service names:
   If you used `fullnameOverride` above the services will be:
   - <release>-redis-front
   - <release>-redis-back

   Use `kubectl get svc -n <namespace> | grep <release>` to list them.

Notes:
- Do NOT commit real passwords to VCS. Use Kubernetes Secrets, SealedSecrets, or external secret managers.
- Check the exact secret key names and value paths for the Bitnami Redis chart version you choose (they can vary between versions).
